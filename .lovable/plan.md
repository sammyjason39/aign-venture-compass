# Currency-aware financial units

## Problem
The financial dashboard always uses Indonesian magnitude suffixes — `jt` (juta/million), `rb` (ribu/thousand) — even when the report currency is USD. So USD values render as `$410 rb`, `$250 rb`, `$1.150 rb`, and unit-economics cards show the full word `ribu`. For USD this should read in English (e.g. `$410K`, `$250K`, `$1.15M`) or no word at all.

## Fix

### 1. Make `unitSuffix` currency-aware (`src/lib/curation/financial-format.ts`)
- Change `unitSuffix(unit)` to `unitSuffix(unit, currency)`.
  - IDR: `juta → "jt"`, `ribu → "rb"`, `penuh → ""` (unchanged).
  - USD: `juta → "M"`, `ribu → "K"`, `penuh → ""`.
- Update `formatMoney` (which already receives currency) to call the currency-aware version. Spacing: keep `$410K` tight for USD letters, keep `Rp410 jt` spaced for IDR. Negative/zero handling stays the same.

### 2. Update chart-note call sites (`src/components/curation/FinancialDashboard.tsx`)
- The two `note={...unitSuffix(unit)...}` usages (Revenue/EBITDA chart and Cash-flow chart) currently call `unitSuffix(unit)`; pass `currency` too so the axis caption reads `$ M · per period` / `$ K · per period` for USD.

### 3. Stop the AI from emitting Indonesian words for USD (`src/lib/curation/financial-eval.server.ts`)
- The unit-economics cards' `unit` strings (e.g. `ribu`) are free-text generated by the AI. Update the system prompt so that when the chosen currency is USD, card `unit` suffixes use English (`K`, `M`, `x`, `mo`, `%`) and never Indonesian words like `ribu`/`juta`. Keep Indonesian (`rb`, `jt`) only for IDR.

## Notes / scope
- Money KPIs and chart captions are formatted at render time from `model.unit + currency`, so existing saved USD dashboards will instantly show the correct `K`/`M` suffix with no re-run needed.
- Only the AI-generated card `unit` text (the `ribu` word on unit-economics cards) is baked into stored data. Existing USD dashboards will keep their old `ribu` text until an admin re-runs AI or edits the card; new generations will be correct. No data migration is planned.
- Purely presentation/formatting changes; no DB schema or scoring logic touched.

## Technical detail
- `UNIT_SUFFIX` becomes a `Record<FinancialCurrency, Record<FinancialUnit, string>>` lookup.
- `unitSuffix` signature gains a required `currency` param; both call sites in `FinancialDashboard.tsx` and the `formatMoney` internal call are updated accordingly.
